A command-line tool that audits Solidity smart contracts for security vulnerabilities, Mantle L2 gas inefficiencies, and ERC-8004 Agent Identity NFT compliance. DETAILED DESCRIPTION (for project page, ~400 words)
----------------------------------------------------
Most Solidity security tools are built for Ethereum mainnet and miss the characteristics that make Mantle L2 different. The Mantle Smart Contract Audit Assistant is the first static analysis tool designed specifically for the Mantle ecosystem.

WHY MANTLE-SPECIFIC RULES MATTER:

1. Gas Model Inversion
On Ethereum, storage is the dominant cost. On Mantle L2, the L1 data posting fee accounts for ~90% of transaction cost — the optimization priorities are inverted. Generic auditors recommend reducing storage; on Mantle, you should minimize calldata bytes. Our 15 gas rules reflect this: G-06 catches string revert messages (40+ bytes of calldata vs 4 bytes for a custom error), G-10 flags string state variables (should be IPFS hashes), and G-15 gives Mantle DA / EigenLayer compression hints.

2. Sequencer Manipulation
Mantle uses a centralized sequencer that can adjust block.timestamp by ±15 seconds and sees all pending transactions before inclusion. Our V-02 rule flags timestamp-dependent logic as HIGH severity with an explicit L2 marker. V-16 flags front-running exposure patterns that are especially dangerous against a centralized sequencer.

3. Finality Assumptions
Mantle is an optimistic rollup with a 7-day fraud proof window. L2 block confirmation does not equal L1 finality. V-15 catches code that assumes cross-chain settlement is immediate — a common and expensive mistake for protocols bridging from Mantle.

4. ERC-8004 Compliance
ERC-8004 is Mantle's Agent Identity NFT standard, deployed on mainnet in February 2026. No other auditing tool checks ERC-8004 compliance. Our 15 rules cover all three registry components: Identity Registry (7 required functions, 2 events), Reputation Registry (7 functions including the owner-exclusion constraint on giveFeedback, 2 events), and Validation Registry (4 functions, 1 event). We also verify EIP-712 signature handling on setAgentWallet.

TECHNICAL APPROACH:

The tool uses a pure-Python Solidity parser (regex-based AST-like analysis — no external parser libraries) combined with a rule engine that produces structured findings with severity levels (CRITICAL through INFO), line numbers, and L2-specific annotations. The static analysis approach means it works without deploying contracts or connecting to any network.

OUTPUT:
- Terminal: color-coded findings table with Rich formatting
- HTML: self-contained dark-theme report with SVG compliance ring, expandable findings, zero external dependencies

ZERO DEPENDENCIES BEYOND TWO PACKAGES:
pip install click rich — that is the entire setup. No API keys. No Docker. No blockchain node. Works on any machine with Python 3.10+.
